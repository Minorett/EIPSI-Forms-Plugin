v1.3.19 - Config ID Stabilization: Shortcode estable para siempre

OBJETIVO:
Corregir arquitectura de generación de config_id para que sea ESTABLE 
(determinístico) en lugar de generar uno nuevo cada vez que se guarda.

PROBLEMA (v1.3.18):
- config_id = 'config_' . $post_id . '_' . time() . '_' . wp_generate_password()
- time() generaba nuevo valor cada save → nuevo config_id → nuevo shortcode
- Rompía persistencia de assignments y manual overrides
- PILAR 1: Analytics URL cambiaba
- PILAR 2: Manual overrides se perdían
- PILAR 3: Distribution stats se fragmentaban

SOLUCIÓN (v1.3.19):
- config_id = 'rct_post_' . intval($post_id) . '_eipsi'
- Config ID determinístico basado SOLO en post_id
- UPDATE si existe, INSERT si no existe
- Shortcode generado UNA SOLA VEZ, nunca cambia
- Backwards compatible con configs antiguos
