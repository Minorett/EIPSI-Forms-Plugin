fix: Likert radio button selection not working

Critical bug fix for EIPSI Campo Likert block where radio buttons
could not be selected. Clicks appeared to have no effect and 
validation errors persisted, making Likert fields unusable.

Root Cause:
JavaScript event handler was using 'click' event with toggle logic
that unchecked radio buttons immediately after browser selected them.
By the time the handler executed, radio state had already changed,
causing the handler to always uncheck the selection.

Solution:
1. Replaced 'click' event listener with 'change' event
2. Removed problematic toggle behavior
3. Simplified validation trigger logic
4. Added 'pointer-events: none' to hidden radio input
5. Enhanced CSS selectors for better keyboard navigation

Changes:
- assets/js/eipsi-forms.js (lines 774-789) - PRIMARY FIX
  * Changed from click to change event
  * Removed toggle logic and setTimeout workarounds
  * Direct validation call on change

- src/blocks/campo-likert/style.scss (lines 81-171)
  * Improved hidden input positioning (1px × 1px)
  * Added pointer-events: none to prevent click conflicts
  * Enhanced focus indicators for keyboard navigation
  * Better CSS selectors (changed + to ~)

- build/style-index.css - Compiled SCSS changes
